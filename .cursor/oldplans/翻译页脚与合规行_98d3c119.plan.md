# 翻译页脚折叠逻辑 + 页脚 Support/Creem 替换 About

## 背景与根因

- `[FooterWithTranslateBehavior.tsx](D:\imppro\translatepdfonline\frontend\src\themes\default\blocks\FooterWithTranslateBehavior.tsx)` 在路径含 `/translate` 时固定 `useState(false)`，页脚默认 `max-h-0`，必须点「Show footer」才展开。
- 业务上「是否该省垂直空间」应由 `[TranslatePageClient.tsx](D:\imppro\translatepdfonline\frontend\src\app\[locale]\(translate)`\translate\TranslatePageClient.tsx) 的状态决定：约 **587–599 行** `!documentId` 为漏斗首页，**602 行起** 为上传后工作台（含预览）。

## 1. 翻译页：按 documentId 控制折叠

**新增**客户端 Context（建议路径 `[src/shared/contexts/translate-footer-workbench.tsx](D:\imppro\translatepdfonline\frontend\src\shared\contexts\translate-footer-workbench.tsx)`）：

- 提供 `workbenchOpen: boolean` 与 `setWorkbenchOpen(next: boolean)`（或仅 setter）。
- 语义：`workbenchOpen === true` 表示当前存在「已打开文档」的工作台（与 `documentId != null` 对齐）。

**在** `[TranslateAppShell.tsx](D:\imppro\translatepdfonline\frontend\src\app\[locale]\(translate)`\TranslateAppShell.tsx) **中用该 Provider 包裹** 外层 `flex min-h-screen flex-col` 的整块内容（需同时包住 `<main>{children}</main>` 与 `{footer}`，否则页脚读不到 Context）。

**在** `TranslatePageClient` **中**：

- `useEffect(() => setWorkbenchOpen(Boolean(documentId)), [documentId, setWorkbenchOpen])`。
- 上传成功、`handleUploaded` 设置 `documentId` 后会自动 `true`；删除文档、`documentId` 清空后自动 `false`。

**改写** `FooterWithTranslateBehavior`：

- 非 `/translate`：行为不变，直接渲染 `children`。
- `/translate`：消费 Context。
  - `workbenchOpen === false`：**不渲染** 顶部「Show footer」细条；**直接完整渲染** `children`（无 `max-h-0`）。
  - `workbenchOpen === true`：保持现有折叠 UI（细条 + `max-h-[80vh]` + 底部 Collapse），最大化预览区高度。

边界：首次进入 `/translate` 在 hydration 前 Context 默认 `false`，页脚先展开，与需求一致；从 URL 恢复任务导致很快出现 `documentId` 时会自动切到折叠，可接受。

## 2. 页脚：Support/Creem 替换 About 列

**数据层**

- 在 `[landing.d.ts](D:\imppro\translatepdfonline\frontend\src\shared\types\blocks\landing.d.ts)` 恢复精简的 `FooterCompliance`（例如 `support_label?`、`support_email`、`payments` 单行文案，或固定英文结构由 i18n 整段提供）。
- 全部 10 个 `[src/config/locale/messages/*/landing.json](D:\imppro\translatepdfonline\frontend\src\config\locale\messages)` 的 `footer`：
  - **删除** `nav`（含 About / PDF Translate / Pricing）。
  - **增加** `compliance`，各语言写清：客服邮箱 `hello@translatepdfonline.com`（mailto）+ Creem 支付说明（与审核表述一致即可）。

**展示层** `[footer.tsx](D:\imppro\translatepdfonline\frontend\src\themes\default\blocks\footer.tsx)`

- 在现有 `md:grid-cols-5` 中，**右侧原 `col-span-3` 导航格**改为：
  - 若存在 `footer.compliance`：渲染一块文案（建议 `text-sm` / `text-muted-foreground`，一行可用 `Support: … | …` 或两行：邮箱 + Creem；邮箱用 `<a href="mailto:…">`）。
  - 若无 `compliance` 且仍有 `footer.nav`：保留原 `nav` 映射（便于其它环境复用，当前项目可全走 compliance）。

站内到 **Translate / Pricing** 的入口依赖顶栏与其它页面链接即可（与「去掉页脚 About 列」一致）。

## 3. 自检

- 打开 `/translate` 无上传：页脚完整可见，无「Show footer」。
- 上传进入工作台：页脚自动折叠，可手动展开/收起。
- 删除文档回到漏斗：页脚再次默认展开。
- 首页/定价等 `(landing)` 路由：页脚右侧为 Support/Creem，无 About 列。
