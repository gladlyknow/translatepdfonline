# 翻译页：页眉页脚点击展开、按钮精简、顶部空白优化

## 1. 页眉/页脚：默认隐藏 + 点击展开，去掉悬停

**现状**  
[HeaderWithTranslateBehavior.tsx](frontend/src/themes/default/blocks/HeaderWithTranslateBehavior.tsx) 与 [FooterWithTranslateBehavior.tsx](frontend/src/themes/default/blocks/FooterWithTranslateBehavior.tsx) 在翻译页通过「窄条 + 悬停」展开：`onMouseEnter` → 展开，`onMouseLeave` → 延迟收起；窄条文案为 `hoverToShowHeader` / `hoverToShowFooter`。

**改动**

- **交互**：去掉 `onMouseEnter` / `onMouseLeave` 及 `scheduleHide` / `hideTimeoutRef`。仅保留 `expanded` 状态，通过**点击窄条上的按钮**切换：未展开时点击 → 展开，已展开时可在页眉/页脚内提供收起按钮，或再次点击同一「展开条」收起（若设计为条可点击整条收起）。
- **窄条**：保留一条固定高度的顶/底条。条内放一个按钮，文案改为「显示页眉」/「显示页脚」（如 `showHeader` / `showFooter`），点击后 `setExpanded(true)`；展开后页眉/页脚区域可带一个「收起」按钮调用 `setExpanded(false)`。
- **文案**：在 [en/translate/home.json](frontend/src/config/locale/messages/en/translate/home.json) 与 [zh/translate/home.json](frontend/src/config/locale/messages/zh/translate/home.json) 中：
  - 新增 `showHeader` / `showFooter`（如 "Show header" / "Show footer"，中文 "显示页眉" / "显示页脚"），
  - 可选保留或移除 `hoverToShowHeader` / `hoverToShowFooter`（若不再使用则删除 key 或保留作备用）。

**涉及文件**

- `frontend/src/themes/default/blocks/HeaderWithTranslateBehavior.tsx`
- `frontend/src/themes/default/blocks/FooterWithTranslateBehavior.tsx`
- `frontend/src/config/locale/messages/en/translate/home.json`
- `frontend/src/config/locale/messages/zh/translate/home.json`

## 2. Start Translation 按钮：文案缩短 + 尺寸缩小

**现状**  
[TranslationForm.tsx](frontend/src/shared/components/translate/TranslationForm.tsx) 提交按钮使用 `t('submit')`。en 为 "Start Translation"，zh 为 "开始翻译"。compact 时按钮已有 `max-w-[11rem]`，视觉仍偏大。

**改动**

- **文案**  
  - [en/translate/translate.json](frontend/src/config/locale/messages/en/translate/translate.json)：`"submit": "Start Translation"` → `"submit": "START"`  
  - [zh/translate/translate.json](frontend/src/config/locale/messages/zh/translate/translate.json)：`"submit": "开始翻译"` → `"submit": "开始"`  
  - 其他语言：当前仅 en/zh 有 `translate/translate.json`；若之后新增 locale，在对应 `translate/translate.json` 中为 `submit` 使用同样风格的简短词（如西语 "Iniciar" 等）即可。
- **按钮尺寸**（仅 compact 模式）：在 `TranslationForm` 中为提交按钮再收紧样式，例如：`max-w-[5rem]` 或 `px-3 py-1.5 text-xs`，使 "START" 更紧凑，与语言选择框视觉平衡。

不新增 key，继续用 `submit`，仅改文案与样式。

## 3. Start Translation 与 History 之间空白：重新设计，必要时居中

**现状**  
[TranslatePageClient.tsx](frontend/src/app/[locale]/(landing)/translate/TranslatePageClient.tsx) 顶部区域（上传 + 表单 + History + 折叠按钮）在 `max-w-[1600px]` 容器内左对齐排列，右侧留白大；与下方 PDF 预览区之间的垂直间距也可能显得空。

**改动思路**

- **优先方案——内容居中**：在保持 `max-w-[1600px]` 的前提下，将顶部控制区内容整体居中，使左右留白对称，减少「左侧一坨、右侧一大块空」的突兀感。
  - 实现：对包含 UploadDropzone、TranslationForm、HistoryPanel、折叠按钮的 flex 容器使用 `justify-center`，或在外层加一层 `flex justify-center`，让内层 `flex flex-wrap` 作为一组居中。注意小屏时仍保持换行与可读性。
- **可选收紧**：若仍觉空，可：
  - 适当减小该区域的 `max-w`（如 1200px），让控制区更集中；或
  - 微调顶部区域与下方 PDF 区之间的 padding/margin，减少垂直空隙。
- **不改变**：History 与 Start 按钮的相对顺序和功能保持不变，仅通过居中和可选宽度/间距优化视觉。

**涉及文件**

- `frontend/src/app/[locale]/(landing)/translate/TranslatePageClient.tsx`（约 414–416 行附近：`mx-auto w-full max-w-[1600px]` 与内部 `flex flex-col gap-3 sm:flex-row sm:items-stretch` 的布局）

## 实施顺序建议

1. 文案与按钮样式（translate.json 的 submit + TranslationForm 按钮 class）  
2. 页眉/页脚改为点击展开（两个 Behavior 组件 + home.json 文案）  
3. 顶部区域居中与间距（TranslatePageClient 布局）

这样可先看到按钮和页眉页脚的变化，再统一收尾布局观感。
