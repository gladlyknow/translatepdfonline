# 前端默认英语、英文 SEO 与品牌图标方案

## 1. 默认语言改为英语

- **[frontend/i18n/config.ts](frontend/i18n/config.ts)**  
  - 将 `defaultLocale` 从 `"zh"` 改为 `"en"`。  
  - `locales` 顺序保持 `["zh", "en", "es"]` 不变（仅影响默认行为与根路径重定向）。
- **[frontend/app/layout.tsx](frontend/app/layout.tsx)**  
  - 将根布局的 `<html lang="zh">` 改为 `<html lang="en">`，与默认语言一致。
- **[frontend/app/[locale]/layout.tsx](frontend/app/[locale]/layout.tsx)**  
  - `loadMessages` 的 fallback 当前为 `dict.zh`；改为 `dict.en`，使未知 locale 时使用英文文案。
- **根路径重定向**  
  - [frontend/app/page.tsx](frontend/app/page.tsx) 已使用 `defaultLocale`，修改 config 后访问 `/` 将自动重定向到 `/en`，无需改此文件。

---

## 2. Title 与 Description 改为英文

- **[frontend/app/layout.tsx](frontend/app/layout.tsx)**  
  - `metadata.title` 改为 `"Online PDF Translation"`。  
  - `metadata.description` 改为英文，例如：  
  `"Translate PDF documents online between English, Chinese and Spanish. Fast and free."`
- **[frontend/app/[locale]/layout.tsx](frontend/app/[locale]/layout.tsx)**  
  - 将同一套 `metadata`（`title` / `description`）改为上述英文，保证所有 locale 下 SEO 与分享时显示一致英文。

## 3. 自定义图标与品牌（translatepdfonline.com）

**现状**：项目已有 [frontend/app/favicon.ico](frontend/app/favicon.ico)，Next.js 会自动使用。

**方案（二选一或组合）**：

**方案 A：文件式图标（推荐，便于替换为设计稿）**

- 在 `app/` 下放置：
  - **favicon.ico**：替换为站点专用图标（如 32×32 或 48×48）。
  - 可选：**icon.png**（如 32×32）、**apple-icon.png**（如 180×180），Next.js 会自动生成对应 `<link>`。
- 图标需你方设计或委托设计后放入上述路径；代码侧无需改 metadata，Next 会按 [Metadata Files 约定](https://nextjs.org/docs/app/api-reference/file-conventions/metadata/app-icons) 自动识别。

**方案 B：代码生成占位图标（无设计稿时先上线）**

- 新增 **app/icon.tsx**，使用 Next.js `ImageResponse`（`next/og`）生成一张简单 PNG：
  - 内容可为品牌首字母 “T”/“TPO” 或简单文档图标 + 主色，尺寸 32×32（及 48×48 若需要）。
- 同时保留或替换 **app/favicon.ico**（.ico 仍建议用文件，因 `ImageResponse` 通常用于 PNG）。
- 之后有正式 logo 时，用方案 A 的文件替换即可。

**品牌与分享（可选）**

- 在 [frontend/app/layout.tsx](frontend/app/layout.tsx) 的 `metadata` 中补充：
  - `metadata.applicationName: "Online PDF Translation"`（或 "Translate PDF Online"）。
  - 若需要社交分享图：在 `app/` 下增加 **opengraph-image.png**（如 1200×630），或在 layout 的 `metadata.openGraph` 中指定 `images`。
- 以上与“图标”独立：图标主要影响浏览器标签/书签，Open Graph 影响社交预览。

**实施建议**

- 若已有或即将有 logo/favicon 设计稿：采用 **方案 A**，在 `app/` 下替换/添加 `favicon.ico`（及可选的 `icon.png`、`apple-icon.png`），并补充上述英文 metadata 与可选 `applicationName`/`openGraph`。
- 若暂无设计稿：先实现 **方案 B** 的 `app/icon.tsx` 占位 + 英文 metadata，再在拿到设计后按方案 A 替换为正式图标文件。
