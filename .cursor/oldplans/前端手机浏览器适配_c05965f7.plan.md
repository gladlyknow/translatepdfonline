# 前端手机浏览器适配方案

## 问题与原因

- **翻译设置无法正常选择语言**：首页在“上传 PDF 后”使用紧凑单行布局的 [TranslationForm](frontend/components/TranslationForm.tsx)（`compact`），内嵌两个 [LanguageSelector](frontend/components/LanguageSelector.tsx) 的 `<select>`。在手机上主要问题有：
  1. **iOS 自动缩放**：`<select>` 使用 `text-sm`（14px）时，聚焦会触发 Safari 放大，体验差且可能“选不准”。
  2. **触控目标过小**：下拉框未设最小高度，低于 44px 不利于手指点击。
  3. **窄屏布局过挤**：紧凑模式为 `flex flex-wrap items-center gap-3`，在窄屏下两个下拉框并排被压得很窄，难以点击和辨认。
- **其他需统一适配**：History 面板在窄屏下定位可能跑出视口；Header 语言/登录按钮触控区域偏小；部分小按钮在触屏上不够友好。

---

## 1. 翻译设置：语言选择可用性（核心）

### 1.1 LanguageSelector

**文件**：[frontend/components/LanguageSelector.tsx](frontend/components/LanguageSelector.tsx)

- **防止 iOS 缩放**：将 `<select>` 的 `text-sm` 改为 `**text-base`**（16px），避免聚焦时 Safari 自动放大。
- **触控目标**：为 `<select>` 增加 `**min-h-[44px]`**，满足约 44px 最小触控高度。
- **可选**：为包裹的 `div` 增加 `**min-w-0`**，避免在 flex 布局中被挤压时撑破或无法收缩。

### 1.2 TranslationForm（紧凑模式）

**文件**：[frontend/components/TranslationForm.tsx](frontend/components/TranslationForm.tsx)

- **小屏改为纵向堆叠**：紧凑模式当前为单行 `flex flex-wrap items-center gap-3`。在窄屏下改为**先纵向再横排**，使两个语言选择器在手机上各占一行、易于点击：
  - 表单根节点 class 改为：`**flex flex-col gap-3 sm:flex-row sm:flex-wrap sm:items-center`**。
- **移动端控件宽度**：在纵向布局下，让语言选择、页码输入、按钮在窄屏时占满可用宽度，例如表单项容器加 `**w-full sm:w-auto`**（或对 LanguageSelector 外层在 compact 模式下用 `w-full sm:w-auto`），避免两个 select 在竖排时仍过窄。
- **提交按钮触控**：紧凑模式下的提交按钮增加 `**min-h-[44px]`**，与无障碍/触控规范一致。

## 2. 首页：翻译区域容器

**文件**：[frontend/app/[locale]/page.tsx](frontend/app/[locale]/page.tsx)

- 包裹 TranslationForm 的 div（约 549–558 行）当前为 `flex flex-[1.5] flex-wrap items-center gap-3 ...`。建议：
  - 增加 `**min-w-0`**，避免在 flex 布局中撑爆宽度。
  - 小屏下可让该块占满一行并给表单留足纵向空间（已由 TranslationForm 的 `flex-col` 负责）；如需可加 `**flex-col sm:flex-row`** 与内层表单一致，或保持现状仅靠表单内部响应式即可，视最终效果二选一。

---

## 3. History 面板：窄屏定位与宽度

**文件**：[frontend/components/HistoryPanel.tsx](frontend/components/HistoryPanel.tsx)

- **现状**：弹层用 `panelRect.left = rect.right - 320`、固定宽度 `w-[320px]`。在视口宽度 ≤320px 或较窄时，面板会超出左边界或右侧露出很少，难以使用。
- **调整**：
  - **定位**：在计算 `panelRect` 时（约 28–30 行），根据视口宽度与按钮位置，将 `left` 限制在可视范围内。例如：
    - 若 `window.innerWidth < 360`，可居中：`left: (window.innerWidth - Math.min(320, window.innerWidth - 16)) / 2`；
    - 或通用：`left = Math.max(8, Math.min(rect.right - 320, window.innerWidth - 328))`，保证左侧至少 8px、右侧至少 8px。
  - **宽度**：弹层 class 增加 `**max-w-[calc(100vw-16px)]`** 或 `**w-[min(320px,calc(100vw-16px))]`**，避免在窄屏下溢出视口。
- **触控**：历史项和“取消/删除”按钮已可点击；若有 `text-[10px]` 的小按钮，可考虑在移动端略增 padding（如 `py-2`）以扩大触控区域，非必须。

---

## 4. Header：触控与窄屏

**文件**：[frontend/components/Header.tsx](frontend/components/Header.tsx)

- **语言切换按钮**：当前 `px-3 py-1 text-xs`，触控区域偏小。为每个 locale 按钮增加 `**min-h-[44px] min-w-[44px]`**（或至少 `py-2` 使高度接近 44px），并保持 `flex items-center justify-center`。
- **“Sign In” / “Get Started”**：为链接容器或链接本身增加 `**min-h-[44px] inline-flex items-center`**（或至少足够 padding），保证约 44px 触控高度。
- **标题**：在极窄屏下标题可被截断，已用 `tracking-tight`；如需可加 `**truncate max-w-[120px] sm:max-w-none`**，避免把右侧按钮挤出视口。

---

## 5. 其他可选与全局

- **Viewport**：Next.js App Router 通常已注入 viewport。若需显式控制，可在 [frontend/app/layout.tsx](frontend/app/layout.tsx) 的 `metadata` 中增加 `viewport: { width: "device-width", initialScale: 1 }`（视 Next 版本而定）；无问题则可不动。
- **登录/注册页**：表单输入框若使用小于 16px 的字体，在 iOS 上可能触发缩放。可对 [frontend/app/[locale]/login/page.tsx](frontend/app/[locale]/login/page.tsx) 和 register 的 `<input>` 使用 `**text-base`**，与 LanguageSelector 一致。
- **首页“删除文档”按钮**：当前 `text-[10px] px-2 py-1.5`，在移动端略小。可选：在小屏用 `**min-h-[44px] py-2`** 或仅增大 padding，不强制。
