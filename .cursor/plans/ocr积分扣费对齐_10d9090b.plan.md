# OCR 积分扣费对齐计划

## 目标
让 OCR 页面与 translate 页面使用同一套积分规则：按页计费（默认 10 积分/页）、余额不足时阻止创建任务并返回统一错误结构、任务成功后再执行幂等扣费并回填任务计费字段。

## 现状与缺口
- OCR 创建接口 [`frontend/src/app/api/ocr/tasks/route.ts`](d:/imppro/translatepdfonline/frontend/src/app/api/ocr/tasks/route.ts) 目前只做文档/语言校验与入队，不做积分预检。
- OCR 流水线未调用 `consumeCredits`，`translation_tasks.creditConsumeId/creditsCharged` 未回填。
- 前端 OCR 启动页 [`frontend/src/app/[locale]/(translate)/ocrtranslator/OcrTranslatePageClient.tsx`](d:/imppro/translatepdfonline/frontend/src/app/[locale]/(translate)/ocrtranslator/OcrTranslatePageClient.tsx) 没有像 translate 表单那样对 402 不足积分做显式处理。

## 改造方案

### 1) 后端：OCR 创建前积分预检（与 translate 同口径）
- 在 [`frontend/src/app/api/ocr/tasks/route.ts`](d:/imppro/translatepdfonline/frontend/src/app/api/ocr/tasks/route.ts) 增加：
  - 读取 `documents.pageCount`（缺失时按现有策略返回可识别错误码，如 `document_pages_required_for_billing`）。
  - 复用 [`frontend/src/shared/lib/translate-billing.ts`](d:/imppro/translatepdfonline/frontend/src/shared/lib/translate-billing.ts) 的开关/单价读取（`isTranslateCreditsEnabled`、`getTranslateCreditsPerPage`）。
  - 对登录用户调用 [`frontend/src/shared/models/credit.ts`](d:/imppro/translatepdfonline/frontend/src/shared/models/credit.ts) 的 `getRemainingCredits` 做余额校验。
  - 余额不足返回 `402 + code=insufficient_credits + need/have/estimated_pages/credits_per_page`（与 translate 保持一致结构）。
  - 创建 OCR 任务时写入 `translation_tasks.creditsEstimated`。

### 2) 后端：OCR 成功后执行幂等扣费
- 在 OCR 完成路径（队列消费完成并标记任务 `completed` 的位置，当前在 OCR 队列执行链中）接入：
  - 条件：`creditsEnabled=true`、`task.userId` 存在、`creditConsumeId` 为空、状态进入最终成功。
  - 调 `consumeCredits(...)`，`transactionScene` 与 translate 保持同规则（避免引入新账本分支）。
  - 成功后回填 `creditConsumeId`、`creditsCharged`；失败回填 `billingError`，不中断主任务成功状态。
- 优先在 OCR 队列主处理文件 [`frontend/src/shared/lib/ocr-queue.ts`](d:/imppro/translatepdfonline/frontend/src/shared/lib/ocr-queue.ts) / 相关完成落库模块中实现，保持一次性事务边界与幂等判断。

### 3) 前端：OCR 启动页补齐积分不足交互
- 在 [`frontend/src/app/[locale]/(translate)/ocrtranslator/OcrTranslatePageClient.tsx`](d:/imppro/translatepdfonline/frontend/src/app/[locale]/(translate)/ocrtranslator/OcrTranslatePageClient.tsx)：
  - 处理 `startOcrTask` 的 `402` 响应，读取 `need/have` 并展示与 translate 一致的不足提示（可复用现有信用不足弹窗组件或等价提示容器）。
  - 对 `document_pages_required_for_billing` 与 `insufficient_credits` 做明确文案映射，避免只显示通用错误。

### 4) API 客户端与错误透传对齐
- 复用现有 [`frontend/src/shared/lib/translate-api.ts`](d:/imppro/translatepdfonline/frontend/src/shared/lib/translate-api.ts) 错误透传（`status/body`），无需新增协议层。
- 仅确认 OCR 分支前端读取 `err.body` 的逻辑与 translate 分支一致。

### 5) 验证与上线检查
- 场景验证：
  - 余额充足：OCR 创建成功，任务完成后扣减 `页数 * 10`。
  - 余额不足：创建阶段即 402 拦截，不入队。
  - 重试/重复回调：不会重复扣费（`creditConsumeId` 幂等）。
  - 访客用户：保持当前策略（不扣或按既定限制），不影响任务稳定性。
- 数据核对：`credit` 账本记录与 `translation_tasks.creditsCharged` 一致。
- 发布前后重点观察 OCR 队列日志与 billingError。
