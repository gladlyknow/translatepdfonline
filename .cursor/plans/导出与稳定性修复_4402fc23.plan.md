# 导出链路与稳定性对齐计划

## 目标
- 修复 `MD...` 导出时缺少转圈标识。
- 解决导出阶段 `Worker exceeded memory limit` 与导出长时间卡住问题。
- 参照 `D:/imppro/onlinepdftranslator` 将当前导出链路收敛为“任务生成 -> 队列执行 -> R2 落盘 -> 签名下载”。
- 彻底清理“无任务空扫”行为，避免资源浪费。
- 将图片缩略图改为 `tpdf.svg`，并将浏览器标签最小图标改为 `LOGO copy.png`，资产放入正式目录。

## 现状与问题点（只读调研）
- 导出前端目前已接入 `exports` API，但状态机会受历史 `pending/processing` 影响，易出现假转圈。
- 导出执行主要在 [D:/imppro/translatepdfonline/frontend/src/shared/lib/ocr-export-queue.ts](D:/imppro/translatepdfonline/frontend/src/shared/lib/ocr-export-queue.ts)，PDF 生成存在整块内存峰值风险（字体嵌入 + 一次性 save）。
- 空扫相关入口在 API 侧仍可被调度：
  - [D:/imppro/translatepdfonline/frontend/src/app/api/ocr/dispatch-pending/route.ts](D:/imppro/translatepdfonline/frontend/src/app/api/ocr/dispatch-pending/route.ts)
  - [D:/imppro/translatepdfonline/frontend/src/app/api/translate/dispatch-pending/route.ts](D:/imppro/translatepdfonline/frontend/src/app/api/translate/dispatch-pending/route.ts)
  - consumer 的 `scheduled` 已 noop，但平台 trigger/外部调度仍可能触发。
- 图标与预览图入口分散在：
  - [D:/imppro/translatepdfonline/frontend/src/app/layout.tsx](D:/imppro/translatepdfonline/frontend/src/app/layout.tsx)
  - [D:/imppro/translatepdfonline/frontend/src/shared/lib/seo.ts](D:/imppro/translatepdfonline/frontend/src/shared/lib/seo.ts)
  - [D:/imppro/translatepdfonline/frontend/src/config/index.ts](D:/imppro/translatepdfonline/frontend/src/config/index.ts)

## 实施步骤

### 1) 前端导出状态细节修复（含 MD 转圈）
- 文件：
  - [D:/imppro/translatepdfonline/frontend/src/shared/ocr-workbench/OcrParseWorkbench.tsx](D:/imppro/translatepdfonline/frontend/src/shared/ocr-workbench/OcrParseWorkbench.tsx)
- 调整：
  - 为 MD/PDF 两个按钮统一状态机（`idle/processing/ready/failed`），`processing` 显示 spinner。
  - 避免历史任务状态导致“默认转圈”：仅“本次触发”保留 processing，历史 pending/processing 不自动转圈。
  - `ready` 后显示下载态，点击实时请求签名 URL 再下载。

### 2) 导出后端链路对齐 online 项目
- 对齐参考：
  - `onlinepdftranslator` 的导出处理与状态推进（任务化导出、ready/fail 终态治理）。
- 当前项目修改点：
  - [D:/imppro/translatepdfonline/frontend/src/app/api/ocr/tasks/[taskId]/exports/route.ts](D:/imppro/translatepdfonline/frontend/src/app/api/ocr/tasks/[taskId]/exports/route.ts)
  - [D:/imppro/translatepdfonline/frontend/src/shared/lib/ocr-export-queue.ts](D:/imppro/translatepdfonline/frontend/src/shared/lib/ocr-export-queue.ts)
  - [D:/imppro/translatepdfonline/frontend/src/shared/models/ocr_task_export.ts](D:/imppro/translatepdfonline/frontend/src/shared/models/ocr_task_export.ts)
- 调整重点：
  - 强化“单次 claim + 终态收敛 + stale fail”机制，避免导出任务长期 pending/processing。
  - 控制导出内存峰值：降低一次性对象并存，分阶段释放中间对象；必要时缩小单次处理批量。
  - 保持下载只走 `downloadUrl=1` 签名链路，禁止前端本地兜底导出。
