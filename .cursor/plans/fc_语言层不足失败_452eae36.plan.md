# FC：语言层过弱时不当作翻译成功

## 背景与根因

- 当前 [`babeldoc_fc/main.py`](d:/imppro/translatepdfonline/babeldoc_fc/main.py) 在 `run_translate_local_with_retries` 成功返回后即上传 PDF 并 [`_notify_completed_callback_with_retry`](d:/imppro/translatepdfonline/babeldoc_fc/main.py) 标记 **completed**，不区分「仅页眉页脚少量段落译完、正文多为 CID/缺字形」。
- BabelDOC 在 [`high_level.py`](d:/imppro/translatepdfonline/tmp/BabelDOC/babeldoc/format/pdf/high_level.py) 把 `valid_char_count_total` 写入 [`TranslateResult.total_valid_character_count`](d:/imppro/translatepdfonline/tmp/BabelDOC/babeldoc/format/pdf/translation_config.py)；可作为「语言层」强度的信号之一。
- [`ILTranslatorLLMOnly`](d:/imppro/translatepdfonline/tmp/BabelDOC/babeldoc/format/pdf/document_il/midend/il_translator_llm_only.py) 在翻译结束处打日志 `Total / Successful / Fallback`（你日志里的 `Total: 3, Successful: 2, Fallback: 1`），但这些 **未挂到 `TranslateResult`**；且 `do_translate` 的 `finally` 会 `cleanup_temp_files()`，**无法在 `translate()` 返回后再读 working 目录里的 tracking JSON**。

## 目标行为

- 在 **上传 R2 与回调 completed 之前** 做校验；不满足则：
  - **不上传**（或不上传为主产物，按现有失败路径即可）；
  - `callback` 发 **`failed`**，并带 **`error_code: scan_detected_use_ocr`**（与现有 [`frontend/src/app/api/translate/callback/route.ts`](d:/imppro/translatepdfonline/frontend/src/app/api/translate/callback/route.ts) / 翻译页 OCR CTA 一致）。
- 用 **环境变量** 控制阈值与总开关，便于线上调参：`BABELDOC_INSUFFICIENT_TEXT_CHECK`（默认 on）、`BABELDOC_MIN_VALID_CHARS_PER_PAGE`、`BABELDOC_MIN_VALID_CHARS_TOTAL`、以及（若接段落统计）`BABELDOC_MIN_OK_RATIO` 等。

## 实现方案（推荐：两段式信号）

### 1）在 `tmp/BabelDOC` 增加轻量统计（推荐，解决漏判）

- 在 [`SharedContextCrossSplitPart`](d:/imppro/translatepdfonline/tmp/BabelDOC/babeldoc/format/pdf/translation_config.py) 增加字段，例如：`paragraph_extractable_total`、`paragraph_llm_total`、`paragraph_llm_ok`、`paragraph_llm_fallback`（命名可缩短）。
- 在 [`ILTranslatorLLMOnly.translate`](d:/imppro/translatepdfonline/tmp/BabelDOC/babeldoc/format/pdf/document_il/midend/il_translator_llm_only.py) 末尾（已有 `total` 可抽取段落数、以及 `self.total_count/ok_count/fallback_count`）写入上述 shared_context。
- 在 [`high_level.py`](d:/imppro/translatepdfonline/tmp/BabelDOC/babeldoc/format/pdf/high_level.py) 填充 `TranslateResult` 时（与 `total_valid_character_count` 同一段 try 块附近），把 shared_context 中的段落统计 **拷贝到 `TranslateResult`**（需在 [`TranslateResult`](d:/imppro/translatepdfonline/tmp/BabelDOC/babeldoc/format/pdf/translation_config.py) 上增加可选属性并 `__str__` 可忽略）。

这样 **不依赖 working 目录**，与现有 pipeline 一致。

### 2）在 `babeldoc_fc` 做门闸（核心逻辑）

- 在 [`run_translate.py`](d:/imppro/translatepdfonline/babeldoc_fc/run_translate.py) 的 `translate(config)` 返回之后、返回 `output_paths` 之前（或由 [`main.py`](d:/imppro/translatepdfonline/babeldoc_fc/main.py) 在拿到 `result` 后统一校验，若 `run_translate` 只返回 paths 则需扩展返回值携带 `TranslateResult` 或上述统计）：
  - 用 **pypdf** 对输入 PDF 求页数（与 main 中 [`_pdf_num_pages`](d:/imppro/translatepdfonline/babeldoc_fc/main.py) 一致，可抽成小函数避免重复）。
  - 计算 `chars_per_page = total_valid_character_count / max(pages, 1)`。
  - **规则示例**（最终数值写进 README，默认偏保守、减少误杀）：
    - `total_valid_character_count` 为 0 或缺失 → 失败（可归 `no_paragraphs` 或统一 `scan_detected_use_ocr`，建议与产品现有文案对齐选其一）。
    - `pages >= 2` 且 `chars_per_page < MIN1` → `scan_detected_use_ocr`。
    - 若有段落统计：`extractable_total` 相对页数过少（如 `extractable_total < pages * K`）或 `ok_ratio = ok / max(llm_total,1) < MIN2` 且 `llm_total >= N` → `scan_detected_use_ocr`（覆盖「只译了 3 段但多页扫描」类案例）。
  - 校验失败：**抛异常**（消息里含稳定关键词），由 main 映射为 `scan_detected_use_ocr`。

### 3）错误码映射

- 在 [`babeldoc_fc/main.py`](d:/imppro/translatepdfonline/babeldoc_fc/main.py) 的 [`_error_code_for_babeldoc_failure`](d:/imppro/translatepdfonline/babeldoc_fc/main.py) 中为新异常/消息增加分支，返回 **`scan_detected_use_ocr`**（或单独 `insufficient_text_layer_ocr` 若你希望前端文案区分；你要求「OCR 建议按钮」则复用 `scan_detected_use_ocr` 成本最低）。

### 4）前端 / Worker

- [`callback/route.ts`](d:/imppro/translatepdfonline/frontend/src/app/api/translate/callback/route.ts) 已支持 `error_code`；若仅用已有码，**通常无需改 UI**。
- 若走同步 `invoke-fc` 路径，核对 [`frontend/src/app/api/translate/invoke-fc.ts`](d:/imppro/translatepdfonline/frontend/src/app/api/translate/invoke-fc.ts) 是否需同步映射（与 callback 一致即可）。

### 5）文档与回归

- 更新 [`babeldoc_fc/README.md`](d:/imppro/translatepdfonline/babeldoc_fc/README.md)：新 env、默认阈值、如何关闭检查。

## 风险与缓解

- **误杀**：纯图表/页数多但合法文本极少的 PDF。缓解：阈值用 env、默认略宽松；提供 `BABELDOC_INSUFFICIENT_TEXT_CHECK=0` 一键关闭。
- **镜像构建**：改了 `tmp/BabelDOC` 后需确保 FC 镜像仍 `pip install -e` 该路径（与现有 [build-babeldoc-fc.yml](d:/imppro/translatepdfonline/.github/workflows/build-babeldoc-fc.yml) 一致）。
