法律合规与数据安全：AGPL-3.0 协议下的 SaaS 策略
BabelDOC 及其衍生项目主要采用 AGPL-3.0 许可证。这是一个极其严格的强传染性开源协议，对商业化 SaaS 有着明确的要求 。   

AGPL-3.0 的“SaaS 补丁”与披露义务
传统的 GPL 仅在软件“分发”时强制要求开源，而 AGPL 规定只要用户通过网络（即 SaaS 模式）与修改后的程序交互，开发者就必须提供源码 。   

平台实施准则：

文件安全与隐私保护
学术 PDF 可能包含未发表的敏感数据。系统应建立“阅后即焚”机制：

自动清理： 翻译完成后的原始文件与翻译件，在 R2 中的保留时间不得超过 48 小时。

静态加固： 后端生成的预签名 GET 链接应限制 IP 访问或设置极短的有效期 。   

传输加密： 全站强制实施 TLS 1.3，并在 R2 层面开启“静态加密”（Encryption at Rest） 。

模块 A：Next.js + R2 直传组件
功能描述： 实现一个基于 shadcn/ui 的拖拽上传区域。要求：

调用 /api/upload/presigned 获取预签名 URL 。   

使用 fetch(method: 'PUT') 进行二进制直传。

通过 R2 元数据传递 userId。

展示分片上传进度条 。   

模块 B：BabelDOC 异步处理 Worker
功能描述： 构建一个基于 Python 3.12 和 Celery 的翻译 Worker。要求：

接收 Redis 中的 job_id 和 R2 object_key 。   

调用 pdf2zh_next.high_level.do_translate_async_stream 。   

集成 DeepSeek-V3 接口，处理 openai_temperature 等配置参数 。   

捕获 TranslationError 并将错误信息持久化至数据库 。   

模块 C：PDF.js 双侧联动查看器
功能描述： 创建一个 React 双窗格 PDF 查看器组件。要求：

基于 react-pdf-viewer 或原生 PDF.js 渲染两侧页面 。   

实现行索引同步滚动算法，识别 data-index 属性 。   

支持画布坐标与 PDF 坐标的互转，用于区域重译功能 。   

非功能性约束（Cursor NFR）
性能： 单页 PDF 解析耗时应控制在 2s 以内。

扩展性： 翻译接口需适配 OpenAI 格式，以便未来切换模型（如 SiliconFlow） 。   

健壮性： 必须处理网络抖动导致的 LLM 响应中断，支持断点续传翻译进度 。   

关于平台可持续性的深度结论
构建基于 BabelDOC 的多语种 PDF 翻译平台，其成功的核心不仅在于算力的堆砌，更在于对科研工作者阅读痛点的深刻理解。通过实现基于段落索引的高精度同步滚动，平台解决了长期以来翻译软件“版式乱、查阅难”的问题。阿里云香港节点与 Cloudflare R2 的组合，在技术层面解决了跨境流量瓶颈与存储成本红线，为平台的商业化盈利留出了充足的空间。
