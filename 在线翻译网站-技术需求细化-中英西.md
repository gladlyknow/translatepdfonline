```python
def build_system_prompt() -> str:
    return """
你是一个专业的 PDF 文档翻译引擎，擅长在保持原文结构的前提下进行高质量翻译。

要求：
1. 按段落逐条翻译，不合并、不拆分。
2. 保留原有 JSON 数组结构，只在 translated 字段中填入译文。
3. 不翻译 LaTeX 公式、代码、文件路径，遇到此类内容时原样保留。
4. 翻译风格保持正式、自然、学术倾向。
输出：
- 严格输出 JSON 数组格式。
- 不要添加解释、前后缀说明。
""".strip()
```

**2）构建 User Prompt**

```python
def build_user_prompt(
    segments: list[IRSegment],
    source_lang: LangCode,
    target_lang: LangCode,
    style: str = "academic"
) -> str:
    # 只挑选需要翻译的段落类型
    payload = [
        {"id": seg.id, "text": seg.text}
        for seg in segments
        if seg.block_type in ("paragraph", "heading", "caption")
    ]

    return f"""
源语言：{source_lang}
目标语言：{target_lang}

翻译要求：
1. 翻译风格：{style}。
2. 数字、专有名词尽量保持一致。
3. 对每个 id 对应的 text 单独翻译，不合并。

下面是待翻译的 JSON 数组：
{payload}
""".strip()
```

**3）批量翻译主流程**

```python
def translate_segments_for_task(task_id: str, doc_id: str, src: LangCode, tgt: LangCode):
    ir = get_ir_for_document(doc_id)
    # 按 block_type 过滤可翻译段落
    translatable = [s for s in ir.segments if s.block_type in ("paragraph", "heading", "caption")]

    system_prompt = build_system_prompt()
    batch_size = 30  # 可配置

    for batch in batched(translatable, batch_size):
        user_prompt = build_user_prompt(batch, src, tgt)
        raw = call_deepseek_v3(
            prompt=user_prompt,
            system_prompt=system_prompt,
            temperature=0.15,
            top_p=0.9,
        )
        translated_segments = parse_translation_response(raw, [s.id for s in batch])

        # 写回 ir
        mapping: Dict[str, str] = {seg.id: seg.translated for seg in translated_segments}
        for seg in ir.segments:
            if seg.id in mapping:
                seg.translated_text = mapping[seg.id]
        # 可分批持久化
        save_ir_with_translations(doc_id, ir)
```

`parse_translation_response` 负责：

- 尝试 `json.loads`
- 校验是否所有 `id` 覆盖
- 出错时记录日志并重试 / 标记任务失败

```python
# run_translation_task (tasks_translate.py) 核心逻辑
def run_translation_task(task_id: str) -> None:
    task = db.query(TranslationTask).get(task_id)
    doc = db.query(Document).get(task.document_id)

    local_path = resolve_staging_path(doc.object_key, doc.filename)  # 从 BABELDOC_STAGING_DIR 解析
    if not local_path:
        _update_task_status(db, task, "failed")

    output_dir = get_task_output_dir(task_id)
    run_translate(

        output_dir=output_dir,

        target_lang=task.target_lang,
        page_range=task.page_range,
    )
    _update_task_status(db, task, "completed")
```

- `run_translate` 定义在 `babeldoc_adapter.py`，内部调用 BabelDOC 的 `high_level.translate`，使用 DeepSeek（OpenAI 兼容）作为翻译后端。
- 需配置 `DEEPSEEK_API_KEY`、`BABELDOC_STAGING_DIR`，并将待翻译 PDF 放入暂存目录（路径需与 `object_key` 或 `filename` 对应）。

### 5.3 伪代码：同步滚动核心逻辑

```ts
function useScrollSync(segments: SegmentView[]) {
  const segmentsById = indexBy(segments, s => s.id);

  return (side: "source" | "target", _scrollPos: number) => {
    const sourceContainer = document.getElementById("pdf-source");
    const targetContainer = document.getElementById("pdf-target");
    if (!sourceContainer || !targetContainer) return;

    const active = findTopVisibleSegment(side === "source" ? sourceContainer : targetContainer);
    if (!active) return;

    const peerContainer = side === "source" ? targetContainer : sourceContainer;
    const peerElem = peerContainer.querySelector<HTMLElement>(`[data-index="${active.id}"]`);
    if (!peerElem) return;

    const ratio = active.visibleRatio;
    const offset = peerElem.offsetTop + ratio * peerElem.offsetHeight;
    peerContainer.scrollTo({ top: offset, behavior: "smooth" });
  };
}
```

`findTopVisibleSegment`：

```ts
function findTopVisibleSegment(container: HTMLElement) {
  const children = Array.from(container.querySelectorAll<HTMLElement>("[data-index]"));
  const viewportTop = container.scrollTop;
  const viewportBottom = viewportTop + container.clientHeight;

  let best: { id: string; visibleRatio: number } | null = null;

  for (const el of children) {
    const rectTop = el.offsetTop;
    const rectBottom = rectTop + el.offsetHeight;

    const visibleTop = Math.max(rectTop, viewportTop);
    const visibleBottom = Math.min(rectBottom, viewportBottom);
    const visibleHeight = visibleBottom - visibleTop;

    if (visibleHeight <= 0) continue;
    const ratio = visibleHeight / el.offsetHeight;

    if (!best || ratio > best.visibleRatio) {
      best = { id: el.dataset.index!, visibleRatio: ratio };
    }
  }
  return best;
}
```

### 5.4 圈选到 PDF 坐标映射（前端）

```ts
function convertDomBoxToPdfBox(
  viewport: any, // PDF.js viewport
  box: { x1: number; y1: number; x2: number; y2: number }, // DOM 相对 page canvas 的坐标
): [number, number, number, number] {
  const [px1, py1] = viewport.convertToPdfPoint(box.x1, box.y1);
  const [px2, py2] = viewport.convertToPdfPoint(box.x2, box.y2);
  return [px1, py1, px2, py2];
}
```

  - 接入 `next-intl`：支持 zh/en/es 三语，`messages/` 下提供对应 JSON，`i18n/routing.ts` 配置 `localePrefix: "as-needed"`。
  - 实现核心组件：`LanguageSelector`、`UploadDropzone`、`TranslationForm`、`PdfViewerPane`（基于 react-pdf + pdfjs-dist）。
  - API 客户端 `lib/api.ts`：封装 `directUpload`、`translate`、`getTask` 等调用。
  - 后端新增 `POST /api/upload/direct`：直接上传 PDF（本地开发用），保存至 `tmp/staging` 或 `BABELDOC_STAGING_DIR`。
  - `next.config.ts` 配置 `rewrites`，将 `/api/*` 代理到后端。
  - 主流程：上传 PDF → 选择源/目标语言 → 创建翻译任务 → 展示任务状态；双栏 PDF 预览为占位（待接入 R2 预签名 URL 与 `GET /api/tasks/{id}/view`）。

- **v0.7 – 自动 Worker + 真正的 BabelDOC 翻译链路**
  - **后端计算链路完善：**
    - `celery_app.py` 中显式导入 `tasks_pdf`、`tasks_translate`，确保 Celery Worker 能正确注册并消费 `pdf.preprocess_pdf` 与 `translate.run_translation_task` 任务。
    - 出于安全与稳定性考虑，暂时关闭 Celery result backend（仅使用 Redis 作为 broker），避免 Redis 结果存储异常影响 HTTP 请求；后续如需任务结果持久化，可通过独立的 `CELERY_RESULT_BACKEND_URL` 重新开启。
    - 新增 `pdf_utils.get_pdf_page_count(path)`（基于 `PyPDF2`），在 `POST /api/upload/direct` 中写入 `documents.page_count`，为后续 MinerU/BabelDOC 结构化预处理留出扩展点。
    - `run_translation_task` 现已完整接入 BabelDOC + DeepSeek：从 `BABELDOC_STAGING_DIR` 解析本地 PDF 路径，调用 `babeldoc_adapter.run_translate(...)`，将翻译后的 PDF 写入 `BABELDOC_OUTPUT_DIR/{task_id}`，并在任务成功/失败时更新 `translation_tasks.status`。
  - **任务视图与下载接口：**
    - 新增 `GET /api/tasks/{task_id}/view`：返回 `TaskView`，包含：
      - `task`：任务基础信息（`TaskDetail`）
      - `document_filename`：原始文档名
      - `outputs`：当前任务生成的输出文件列表（基于本地 BabelDOC 输出目录扫描 `*.pdf`，后续可替换为 R2 预签名 URL）。
    - 新增 `GET /api/tasks/{task_id}/files/{filename}`：通过 `FileResponse` 代理本地翻译结果 PDF 下载，并做路径遍历防护；生产环境中建议替换为 R2 的预签名下载地址。
  - **前端任务状态与结果展示：**
    - `app/[locale]/page.tsx` 中增加任务状态轮询逻辑：在获得 `taskId` 后每 2 秒调用 `GET /api/tasks/{task_id}`，自动将状态从 `queued` → `processing` → `completed/failed` 显示在「3. 任务状态」区域。
    - 当状态变为 `completed` 时，前端调用 `GET /api/tasks/{task_id}/view`，展示「翻译结果」下载区块（多个输出文件时以列表形式展示）。
    - `PdfViewerPane` 仍基于 `react-pdf`，后续会将原文/译文输出 PDF 的 URL 接入双栏预览，实现真正的 PDF 对比阅读模式。
  - **本地开发启动一致性：**
    - 通过根目录 `start-dev.ps1` + `frontend/package.json` 中的 `dev:all`，实现一条命令同时启动：
      - `backend`：FastAPI + Uvicorn（带 `.env` / `.env.local` 热重载）
      - `worker`：Celery Worker（Windows 下使用 `solo` 池）
      - `frontend`：Next.js 开发服务器（Turbopack）
    - 技术文档在第 4 章、附录中补充本地开发启动脚本与环境要求说明，确保生产/开发配置行为可预测。
